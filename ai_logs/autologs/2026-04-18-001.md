### 📝 AI 辅助开发自动记录 (Auto-Log)

- **[Task]**: 指导团队成员完成 AIAD 项目第一轮开发环境的搭建、MediaCrawler 基础抓取验证、以及向 Git 仓库提交首次 PR 的完整流程。
- **[Prompt]**: 用户询问如何基于现有 MediaCrawler 代码库进行环境配置、运行测试并生成可提交的 Pull Request，同时要求严格遵守 Auto-Log 记录规范。
- **[Rules Applied]**: 
  1. 强制开发日志记录规则（输出末尾附加 Auto-Log 模块并写盘）。
  2. Conda 环境强制使用规则（指定 `E:\AIAD\.conda\mediacrawler\python.exe`）。
  3. 数据获取规范（代码含 try-except、随机 UA、延时、logging 替换 print）。
  4. 代码输出格式要求（Python 3.10+ 及中文注释）。
- **[Skills Used]**: 
  - Python 3.10, asyncio, Playwright, logging, Conda 虚拟环境管理。
  - Git 基础操作（clone, branch, add, commit, push）。
- **[Dev Process/Iterations]**: 
  1. 分析用户所在团队的技术栈约束（双 Conda 环境、MediaCrawler、Playwright）。
  2. 给出精确的命令行步骤以解决 Playwright 下载失败问题（使用国内镜像）。
  3. 提供一个符合所有强制规范的测试脚本 `test_fetch.py`，用于验证抓取链路。
  4. 生成符合文件名规范的 Auto-Log 文件内容供用户直接保存。
