# AI驱动的广告嵌入式植入项目 - 路线规划与8人分工指南

## 1. 项目背景与当前进展
**核心目标**：通过分析小红书评论区和图文/视频内容，利用AI生成贴合语境、自然且高吸引力的植入式广告评论。
**当前进展**：已具备通过MediaCrawler爬取小红书评论、图片和视频的能力，但整体应用场景未构建，团队分工尚未明确。
**技术栈**：MediaCrawler (爬虫), Langgraph (Agent编排), Chroma (向量数据库), Vue3 (前端开发), Figma (前端设计), AI大模型 (内容生成与多模态分析)。

## 2. 项目全景实现路线图 (Implementation Route)
整个项目从数据获取到最终展示，工作流可以分为以下五个阶段：
- **阶段一：数据采集与预处理 (MediaCrawler)**
  - 抓取小红书目标帖子（图文/视频）及相关评论区数据。
  - 数据清洗脱敏，结构化存储（分离文本、图片、视频元数据）。
- **阶段二：多模态内容理解与知识库构建 (AI + Chroma)**
  - 利用视觉/音频模型分析帖子内容，提取场景、氛围、出现的商品等特征。
  - 利用NLP分析现有评论的情感倾向、用户痛点和流行语境。
  - 将分析结果与历史高转化率广告文本向量化，存入Chroma数据库。
- **阶段三：智能体图谱编排 (Langgraph)**
  - 将各单一功能的AI模块封装为Agent节点。
  - 通过Langgraph定义各节点间的数据流转逻辑（如：爬取结束 -> 触发多模态分析 -> 检索Chroma -> 生成广告 -> 审核）。
- **阶段四：广告文案生成与审核 (AI)**
  - 结合帖子语境特征和Chroma检索到的优秀案例，生成贴合当前评论区的植入式广告。
- **阶段五：前端场景展示 (Figma + Vue3)**
  - Figma设计用户交互原型：用户输入帖子URL -> 系统展示分析过程 -> 呈现多条生成的优质广告评论供选择。
  - Vue3开发实现交互界面，并与后端Langgraph工作流对接。

## 3. 8人团队详细分工与智能体 (Agent) 管理
本项目正好有 **8个人**，我们采用“**1人负责管理/开发1个核心智能体(节点)**”的模式。每个人不仅负责该Agent底层的技术实现，还负责其对应的Prompt调优、数据接口设计及功能迭代。

### 🤖 Agent 1: 数据采集与清洗智能体 (Data Harvester Agent)
- **负责人员**：队员1
- **核心技术**：MediaCrawler, Python, 数据清洗
- **具体工作**：
  1. 维护和优化现有的MediaCrawler爬虫代码，处理小红书反爬策略（如代理池、延时控制）。
  2. 接收前端传入的目标帖子链接，自动触发爬取。
  3. 将爬取到的非结构化数据清洗、格式化为标准JSON（分离图片/视频流、文案、评论列表），传递给下游Agent。

### 🤖 Agent 2: 多模态视觉理解智能体 (Vision Analyst Agent)
- **负责人员**：队员2
- **核心技术**：视觉大模型 (如GPT-4V / Qwen-VL), 图像处理
- **具体工作**：
  1. 接收Agent 1传来的图片或视频帧。
  2. 编写多模态Prompt，让AI识别图片中的使用场景、商品类别、人物情绪、整体氛围（Vibe）。
  3. 输出结构化的视觉语境报告（例如：场景=海边露营，氛围=轻松休闲），供文案生成参考。

### 🤖 Agent 3: 评论区语境与情感智能体 (Context NLP Agent)
- **负责人员**：队员3
- **核心技术**：LLM, 情感分析, NLP
- **具体工作**：
  1. 接收Agent 1传来的原始评论区数据。
  2. 分析当前评论区的主要情感导向、用户痛点（如“求链接”、“防晒效果好吗”）以及语言风格（如是否包含大量Emoji、网感词汇）。
  3. 提炼出“最易切入的广告植入角度”。

### 🤖 Agent 4: 记忆与检索智能体 (RAG & Memory Agent)
- **负责人员**：队员4
- **核心技术**：ChromaDB, Embedding模型, RAG机制
- **具体工作**：
  1. 搭建和维护Chroma向量数据库。
  2. 将历史成功的广告评论、产品知识库进行向量化存储。
  3. 在生成广告前，根据Agent 2和Agent 3提取的语境特征，在Chroma中进行语义检索（相似召回），为生成提供Few-Shot参考。

### 🤖 Agent 5: 核心文案生成智能体 (Creative Copywriter Agent)
- **负责人员**：队员5
- **核心技术**：核心LLM, Prompt Engineering (高级提示词工程)
- **具体工作**：
  1. 聚合前置节点的所有信息（视觉报告、评论语境、Chroma检索出的参考案例、需植入的广告产品信息）。
  2. 精心设计System Prompt，让AI生成多条不同风格（如：测评风、随口安利风、科普风）的植入式评论。
  3. 确保广告植入自然、“不显山露水”、极具吸引力。

### 🤖 Agent 6: 工作流编排与路由智能体 (Langgraph Orchestrator Agent)
- **负责人员**：队员6
- **核心技术**：Langgraph, 状态管理, Python后端API
- **具体工作**：
  1. 作为整个后端的大脑，使用Langgraph将前5个Agent编排成一张有向图（Graph）。
  2. 定义全局State（状态字典），处理各Agent之间的数据流转机制。
  3. 设计条件路由（Conditional Edges），例如：如果生成的文案被内部评估打分过低，则自动打回给Agent 5重写。
  4. 暴露后端API接口供前端调用。

### 🤖 Agent 7: 场景交互设计智能体 (UX/UI Design Agent)
- **负责人员**：队员7
- **核心技术**：Figma, 用户体验设计, 交互逻辑
- **具体工作**：
  1. 将单纯的技术链路转化为一个“好用的SaaS工具场景”。
  2. 使用Figma设计完整的前端页面：首页输入框（粘贴小红书链接+输入要推广的商品）、数据分析加载动画、多模态分析结果仪表盘、最终广告评论挑选界面。
  3. 梳理用户操作路径，确保界面对没有技术背景的广告优化师同样友好。

### 🤖 Agent 8: 前端开发与对接智能体 (Vue3 Frontend Agent)
- **负责人员**：队员8
- **核心技术**：Vue3, TypeScript, TailwindCSS / Element-Plus, Axios
- **具体工作**：
  1. 将队员7的Figma设计稿1:1还原为Vue3前端代码。
  2. 与队员6提供的Langgraph API进行数据对接，处理长轮询或WebSocket（由于Agent工作流耗时较长，需实现实时的进度条/状态更新显示）。
  3. 完成前端界面的最终打包和部署准备。

## 4. 团队协同工作流演示
1. **用户（广告主）** 在 **队员8** 开发的前端页面输入小红书帖子链接和需要植入的产品描述。
2. 请求发送给 **队员6** 的 Langgraph 编排器。
3. **队员1** 的爬虫Agent开始运作，抓取素材；
4. 并行触发：**队员2** 分析视觉素材，**队员3** 分析评论语境；
5. **队员4** 根据2和3的分析结果，从Chroma提取历史最佳对标文案；
6. **队员5** 的生成Agent综合所有信息，产出几条完美的植入评论；
7. 最终结果返回前端，用户在 **队员7** 设计的精美UI中挑选并复制最满意的评论。
